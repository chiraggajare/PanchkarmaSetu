## 🚀 Quick Summary

### 1. Life Cycle Model
**Hybrid (Waterfall + Agile)**  
Waterfall was used for the database schema and project architecture (predictable, healthcare domain). Agile sprints were used for iterative feature development across 6 sprints.

### 2. Team & Scenario
4-member team with roles: Project Lead, Backend Dev, Frontend Dev, QA/Docs Lead.  
Risk analysis for each member's sudden departure, with mitigation strategies.

### 3. Risk Management — Top 3 Risks
| Priority | Risk | Category | Score |
|----------|------|----------|-------|
| 🔴 1st | Database schema incompatibility | Technical | 20/25 |
| 🟠 2nd | Feature scope creep | Schedule | 16/25 |
| 🟡 3rd | Single point of failure (backend) | Resource | 15/25 |

### 4. Leadership Styles
- **Directive:** Used during critical bug fixes and migration failures — clear, immediate instructions.
- **Collaborative:** Used during sprint planning and architectural design decisions — consensus-driven.

### 5. Gantt Chart
15-week schedule covering Waterfall foundation (W1-W2), 6 Agile sprints (W3-W14), and Waterfall closure (W15).
