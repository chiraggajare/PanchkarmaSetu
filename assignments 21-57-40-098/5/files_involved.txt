# Assignment 5: User Authentication and Role-Based Access Control

## Description
This assignment covers setting up the Django authentication system, implementing logic for customized user roles, and restricting view access based on those roles.

## Files Involved

### 1. User Authentication
*   **core/views.py**: `login_view`, `logout_view`, and `signup` functions.
*   **panchkarma_setu/settings.py**: `AUTH_USER_MODEL`, `LOGIN_REDIRECT_URL`, and `LOGOUT_REDIRECT_URL` configurations.

### 2. Role-Based Logic
*   **core/models.py**: `User` model with `role` field definitions (Patient, Therapist, Centre Head).
*   **core/views.py**: View logic using `request.user.role` to redirect users to their respective portals.

### 3. View Restrictions
*   **core/views.py**: Extensive application of the `@login_required` decorator to protect clinical dashboards.

### 4. Template Security
*   **templates/base.html**: Conditional rendering using `{% if user.is_authenticated %}` to show or hide the logout link and dashboard navigation.

## Key Code Snippets

### Role-Based Access (core/views.py)
```python
@login_required
def dashboard(request):
    if request.user.role == 'patient':
        return render(request, 'core/patient_dashboard.html', context)
    elif request.user.role == 'therapist':
        # ... therapist logic ...
```

### View Protection (core/views.py)
```python
@login_required
def user_profile(request):
    # This view is only accessible if a session is active
    return render(request, 'core/profile.html')
```
